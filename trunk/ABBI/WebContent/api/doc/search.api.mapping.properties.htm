search.api.mapping.properties

This file lives in the WEB-INF folder of the application, and tells the front-end servlet:

API
The name of the class to use for the back end (org.siframework.abbi.api.xds.XDSImpl)
feed
A number of properties to configure what appears in the atom feed
title
The feed title
subtitle
The feed subtitle
icon
The URL to use for the feed icon
logo
The URL to use for the feed logo
rights
A statement of rights
Sample

Content in this feed is provided to receiving applications as authorized by the patient or authorized representative. The information contained herein may contain sensitive patient information, individually identifiable data or other protected health information, and should be protected by adequate security mechanism. The provider of this service communicates this information securely. Responsibility for securing this information after it has been delivered through this service is left to the receiving application, and/or the patient or their authorized representative.

generator
A string describing the feed generator: ABBI Prototype v0.1
mimeType
These parameters provide shortcuts for longer mimeType in the API. The front end will translate parameters using these shortcuts to the specified values before passing them to the back end.
xml
text/xml
json
application/json
pdf
application/json
plain
text/plain
text
text/plain
html
text/html
format
These parameters map from ABBI specified short format codes to longer strings providing a comma separated list of codes which the back-end should search for. There should be values for each of the following:
CCD
CCDA
CCR
class
These parameters provide mappings from ABBI specified class codes to class codes used by the back end. Mappings should be provided for each of the following values:
Summaries
Class code values for CCD (1.0 or 1.1) or CCR Patient Summaries
HandP
History and Physical Notes
Consult
Consult Notes
Discharge
Discharge Summaries
Operative
Operative Notes
Procedure
Procedure Notes
Progress
Progress Notes
Imaging
Imaging Reports
Lab
Lab Results

Back End Configuration

This properties in this file are also used to configure the back end. The XDS back end uses these properties to configure the registry, repostories, and audit configuration. Back-ends are free to define properties that do not overlap with properties used to configure the front-end.

registry
The URL of the registry (used by the back-end)
source
The source patient identifier for the registry (used by the back end)
